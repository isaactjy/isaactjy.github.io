Description of each feature (12 in total)¶

  • datetime - hourly date + timestamp
  • season - 1 = spring, 2 = summer, 3 = fall, 4 = winter
  • holiday - whether the day is considered a holiday
  • workingday - whether the day is neither a weekend nor holiday
  • weather -
  • 1: Clear, Few clouds, Partly cloudy, Partly cloudy
  • 2: Mist + Cloudy, Mist + Broken clouds, Mist + Few clouds, Mist
  • 3: Light Snow, Light Rain + Thunderstorm + Scattered clouds, Light Rain + Scattered clouds
  • 4: Heavy Rain + Ice Pallets + Thunderstorm + Mist, Snow + Fog
  • temp - temperature in Celsius
  • atemp - "feels like" temperature in Celsius
  • humidity - relative humidity
  • windspeed - wind speed
  • casual - number of non-registered user rentals initiated
  • registered - number of registered user rentals initiated
  • count - number of total rentals

To evaluate our model, we will use root mean squared logarithmic error.

In [28]:
def rmsle(y1, y2):
    log1 = np.array([np.log(v + 1) for v in y1])
    log2 = np.array([np.log(v + 1) for v in y2])
    calc = (log1 - log2) ** 2
    return np.sqrt(np.mean(calc))

Conclusion¶

Our basic linear regression model did not perform nearly as well as our random forest regression. This could be attributed to the complex dependencies of the data i.e demand is not linearly dependent on the predictors.
